# 三类场景 Worked Examples

## 加载时机

当需要用真实或高保真样例演示如何把 PRD 方法论转成具体 Skill 时加载。

## 场景 A：内容创作者图文卡片 Skill

### 真实输入

一篇 3000 字观点文章、作者头像、两张产品截图，目标平台是小红书优先，公众号可复用。

### 基线失败

无 Skill 时，Agent 容易只做摘要或泛泛海报提示词；常见错误是紫色渐变、标题过长、截图裁切掉关键信息、连续同版式、文字覆盖人物脸部。

### Skill Brief

```markdown
# Skill Brief

## 名称
social-card-maker

## 一句话定义
把文章、观点或案例转成适合社交平台传播的竖版图文卡片。

## 目标用户
内容创作者、产品运营、独立开发者、活动策划。

## 输入
文章正文、目标平台、图片/截图、品牌色或参考风格。

## 输出
3:4 竖版卡片方案、每张卡片文案、版式选择、素材使用说明、导出检查清单。

## 不适用场景
用户只想摘要文章；用户只要普通长图；用户需要自动发布到平台。
```

### Gotcha

文字不得覆盖人物五官、产品关键 UI 或截图核心数据；没有真实素材时才允许 AI 图兜底。

### 路由用例

- 正例：把这篇产品复盘文章做成 6 张小红书图文卡片。
- 反例：帮我总结这篇文章的三个核心观点。
- 近邻：帮我把这篇文章做成一页公众号长图。

## 场景 B：企业运营 SOP Skill

### 真实输入

客户等级、跟进节奏、CRM 字段、敏感场景升级规则、禁用承诺话术。

### 基线失败

无 Skill 时，Agent 容易机械套话、漏记 CRM 字段、对高风险客户不升级、对结果做过度承诺。

### Skill Brief

```markdown
# Skill Brief

## 名称
customer-followup-sop

## 一句话定义
把客户分层、跟进节奏和记录标准转成 Agent 可执行的运营 SOP。

## 目标用户
客服、客户成功、销售运营、社群运营。

## 输入
客户状态、历史沟通、客户等级、CRM 字段和升级规则。

## 输出
下一步动作、话术建议、CRM 记录字段、升级判断和风险提示。

## 不适用场景
用户只想润色一句话；用户要求绕过合规；用户需要系统自动发消息但没有授权。
```

### Gotcha

高风险用户必须升级人工处理，不得用普通模板继续推进；CRM 必填字段缺失时必须补问。

### 路由用例

- 正例：把这套客户跟进 SOP 做成一个 Agent Skill，能判断升级和写 CRM 记录。
- 反例：把这句话润色得更礼貌。
- 近邻：帮我写一份给新人看的客服培训文档。

## 场景 C：工程开发流程 Skill

### 真实输入

一个功能需求、代码仓库、测试命令、代码风格约束和 PR checklist。

### 基线失败

无 Skill 时，Agent 容易顺手重构、忽略项目规则、不跑测试、只口头声称验证、覆盖用户已有改动。

### Skill Brief

```markdown
# Skill Brief

## 名称
team-dev-workflow

## 一句话定义
让 Agent 按团队标准完成需求分析、计划、实现、测试和审查。

## 目标用户
工程师、测试、技术负责人、代码协作团队。

## 输入
需求、仓库、项目规则、测试命令、验收标准。

## 输出
实现计划、最小改动、测试结果、风险说明和交付总结。

## 不适用场景
用户只问概念；用户只要代码片段；当前仓库没有修改权限。
```

### Gotcha

没有运行验证命令或没有记录失败原因时，不得说“已完成”；不要修改与请求无关的文件。

### 路由用例

- 正例：把我们团队的开发流程沉淀成一个 Skill，要求先读规则、先计划、跑测试。
- 反例：解释一下单元测试是什么。
- 近邻：帮我给这个仓库写 AGENTS.md。
