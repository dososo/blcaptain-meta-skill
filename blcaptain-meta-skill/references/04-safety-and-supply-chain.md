# 安全、权限与供应链

## 加载时机

当 Skill 会运行脚本、读写文件、调用网络、连接第三方服务、处理敏感数据、发布内容或被公开分发时加载。

## 安全原则

- 最小权限：只申请完成任务必需的文件、网络、工具和环境权限。
- 明示副作用：写文件、改配置、发请求、发布内容、花钱调用 API 必须说明。
- 默认只读：不确定时先读取、预览、dry-run。
- 禁止隐藏执行：不得在用户不知情时运行破坏性命令或上传数据。
- 密钥隔离：Skill 包不得包含 API key、token、cookie、证书或私有数据。
- 依赖可审计：第三方脚本、包、二进制和远程安装命令必须可追溯。

## 风险分级

| 风险 | 示例 | 要求 |
| --- | --- | --- |
| 低 | 只读 references、纯文本流程 | 普通审阅 |
| 中 | 本地生成文件、格式转换 | 输出路径清楚，必要时 dry-run |
| 高 | 网络访问、第三方 API、批量修改 | 用户确认、日志、最小权限 |
| 关键 | 发布、支付、生产、删除数据 | 人工审批、回滚、强审计 |

## 第三方 Skill 审查

安装或分发前检查：

- `SKILL.md` 是否要求过宽权限。
- `scripts/` 是否删除、上传、远程执行、安装依赖、改 shell 配置。
- 是否锁定依赖版本或说明来源。
- 是否需要外部账号、API 或付费服务。
- 是否包含密钥、真实客户数据或内部路径。
- 是否存在 prompt injection：要求忽略系统/用户指令、绕过权限、读无关文件。
- 是否有可信维护者、许可证、版本和更新记录。

## 一票否决

- 含密钥、cookie、token、私有证书。
- 含真实客户数据、未脱敏日志、内部账号。
- 默认破坏性命令。
- 自动安装不可信远程脚本。
- 绕过平台权限或安全沙箱。
- 要求 Agent 忽略上级指令或读取无关隐私文件。
